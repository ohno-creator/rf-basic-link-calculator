# 視覚回帰テスト運用ランブック (Visual Regression Runbook)

**作成日時**: 2026-07-04
**検証担当**: Antigravity (調査・視覚検証枠)
**ステータス**: 新規運用手順として確定

---

## 1. 視覚回帰テストの概要

RF Basic Link Calculator では、UI改修（UX-1/UX-2）に伴う「意図しないUI崩れ」や「アクセシビリティ影響」を機械的・自動的に検知するため、ビジュアル回帰テストを導入しています。

- **対象**: 全25ページ（Homeページ + 24種類の計算ツールページ）
- **ビューポート**:
  1. **Desktop**: 1440 × 900
  2. **Mobile**: 375 × 812
  合計 **50枚** の基準画像と比較テストを行います。
- **実行条件**: 環境変数 `TEST_VISUAL=true` が設定されている時のみ実行されます。

---

## 2. 【最重要】macOS/Windows ローカルで基準スナップショットを生成してはいけない理由

ビジュアル回帰テストにおける基準画像（Golden Images）は、`e2e/*-snapshots/` ディレクトリ内に `-linux.png` というサフィックス付きでコミットされています。

> [!WARNING]
> **macOS や Windows ローカル環境で `npx playwright test --update-snapshots` を実行し、生成された画像を Git にコミットしてはいけません。**

### 理由 (フォントレンダラの違い)
OSやブラウザが搭載するフォントラスタライザ（アンチエイリアシングの処理方式や、使用されるフォントファミリーの実体）が Linux/CI 環境と異なるため、ローカルで更新された画像をプッシュすると、**GitHub Actions (Linux) 上で数ピクセルのアンチエイリアスのズレ（赤色差分）が全ページで検出され、CI ゲートが100%失敗します。**

ローカル環境でのテスト実行は、あくまで「テストコードの構文・ロジックの動作確認」までに留め、スナップショット比較結果の正真性判定はすべて **Linux/CI** に任せる必要があります。

---

## 3. UI 改修で「意図的」にデザインが変更される場合の運用フロー

意図的にUIレイアウトを変更し、スナップショットの基準画像を更新する必要がある場合は、以下の手順に従って GitHub Actions (Linux) で生成された正当な画像をリポジトリにコミットバックしてください。

### ステップ 1: 変更のプッシュと CI 失敗の確認
1. デザインを変更したソースコードをブランチにコミットしてプッシュします。
2. GitHub Actions CI 上で `deploy` ジョブ（または `visual` ジョブ）が走り、ビジュアルテストで画像の不一致が検出されて「テスト失敗 (Failed)」になることを確認します。

### ステップ 2: 差分画像 (Diff) の視覚的レビュー
1. 失敗した Actions Run の詳細画面にアクセスします。
2. 画面下部の **Artifacts** セクションから、出力された `playwright-report` (または `test-results`) の zip アーカイブをダウンロードします。
3. ダウンロードした HTML レポートを開くか、`*-diff.png` を開き、**「赤くなっている差分が、自身が加えた意図通りの変更であるか」** を視覚的に検証します。
   - **意図通りである場合**: ステップ 3 に進みます。
   - **意図しない箇所の崩れやフォントずれが混入している場合**: コードを修正し、再度プッシュしてステップ 1 からやり直します。

### ステップ 3: CI による基準画像の更新とコミットバック
GitHub Actions 上で最新の Linux 用スナップショットを自動生成し、ブランチにプッシュバックさせます。

1. GitHub Actions のワークフロー一覧から `Update Visual Snapshots on CI`（または手動実行可能なワークフロー）を選択します。
2. 実行対象のブランチ名（例: `track/ux1-ui-redesign`）を選択し、**Run workflow** を実行します。
3. ワークフローが完了すると、自動的に `-linux.png` 基準画像が更新され、対象ブランチへ `chore(visual): update visual snapshots on Linux CI` というコミットが自動プッシュバックされます。
4. ローカル環境で `git pull` を実行し、更新された基準画像をローカルに取り込みます。

---

## 4. 日常のトラブルシューティングとコマンド一覧

### ローカルでの検証テストの実行 (画像比較は行わず、動作のみ確認)
```bash
TEST_VISUAL=true npx playwright test visual.spec.ts
```

### CI での強制手動実行
GitHub リポジトリの Actions タブから `Deploy GitHub Pages` を `workflow_dispatch` で手動トリガーして実行することができます。
