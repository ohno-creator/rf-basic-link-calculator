## 3. 要判断ポイントの評価と申し送り

### ① 非判定値（換算結果）の neutral 寄せ（要修正）
- **評価**:
  UIリデザイン基準（UI Redesign Plan §2.1「非判定値はneutral化」）に基づくと、合否判定のない純粋な単位換算結果（`dBm`, `mW`, `W`, `dBd換算`, `ERP`）は、すべて `tone="neutral"` (灰色) に統一すべきです。現在は `tone="primary"` (青色) で表示されており、法的基準適合を伴う最重要の `EIRP` (緑色, tone="success") を視覚的に目立たせる設計になっていません。
- **申し送り（Codex / Claude 宛）**:
  [DbmConverterPanel.tsx](file:///Users/pc141/Documents/RF%20Basic%20Link%20Calculator/src/app/tools/_components/DbmConverterPanel.tsx) 内の以下の `MetricCard` について、`tone` を **`tone="neutral"`** に変更してください。
  - `dBm` (140行目付近)
  - `mW` (146行目付近)
  - `W` (152行目付近)
  - `dBd換算` (225行目付近)
  - `ERP` (248行目付近)
  *(EIRP の `tone="success"` および アンテナ端子電力の `tone="neutral"` はそのままで問題ありません)*
