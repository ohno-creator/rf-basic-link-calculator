# Antigravity 発注 — RF学習クエスト「選択肢ランダム化」の実機検証

**対象**: `feature/initial-rf-basic-link-calculator` 最新（コミット `merge(quest): 選択肢の毎回ランダム化` 以降）
**ページ**: `/tools/rf-learning-quest`
**目的**: 選択肢の並びが「意図どおりランダム」かをブラウザ実機で検証する。コード変更は不要（検証のみ）。

## 実装仕様（これが期待挙動）

| # | 場面 | 期待する並び |
|---|---|---|
| S1 | ページ初期表示（未クリック） | lesson.idシードの固定順（リロードしても同じ・チラつきなし） |
| S2 | 問題を**開いた瞬間** | 毎回ランダム（開き直すたびに変わってよい） |
| S3 | **回答した直後**（正誤表示中） | 並びは固定（選んだ選択肢がずれない） |
| S4 | **再挑戦**（もう一度ボタン等で未回答に戻す） | 再ランダム |
| S5 | 別の問題へ移動→戻る | 再ランダム |
| S6 | 認定試験モード | 受験開始ごとにランダム・同一受験中は固定 |

背景: 出題データは994/1000問で「正解が選択肢配列の先頭」。表示側シャッフルが唯一の防衛線。

## 検証手順

### A. 再シャッフルの確認（S2/S4/S5）
1. 任意のモード（例: 入門）で STAGE 1 を開き、4択の並びをメモ（選択肢の先頭数文字でよい）。
2. 別のSTAGEへ移動して戻る → 並びが**変わることがある**か（4択の再ランダムなので1/24で同一もありうる。3回開き直して3回とも完全同一なら不合格）。
3. わざと誤答→「もう一度」→ 並びが再ランダムされるか。

### B. 回答中の固定（S3）
1. 任意の問題で選択肢をクリック → 正誤フィードバック表示中に**並びが動かない**こと（選んだ項目のハイライト位置がずれない）。

### C. 正解位置の分散（統計チェック）
1. 任意のモードで連続する10問に回答し、**正解だった選択肢が上から何番目だったか**を記録する（回答後は正解がハイライトされるので位置が分かる）。
2. 判定: 10問すべて同じ位置なら不合格（確率 (1/4)^9 ≒ 0.0004%）。3種類以上の位置が出現していれば合格。
3. 余力があれば別モードでも10問（計20問）実施。

### D. 初期表示の品質（S1）
1. ページを**リロード**して最初に見える問題カード（あれば）の選択肢が、描画直後に**入れ替わって見えない**こと（チラつき・レイアウトシフトなし）。
2. リロード2回で初期並びが同一であること（シード順＝再現性）。

### E. 認定試験（S6）
1. 認定試験を開始→最初の3問の並びをメモ→ブラウザバック等で中断→**新しく受験を開始**→同じ問題が出たら並びが変わりうることを確認（受験ごとのシード）。
