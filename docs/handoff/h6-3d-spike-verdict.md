# H6 3D技術検証スパイク — 判定: 予算超過につき2.5D経路を採用

**ブランチ**: `track/h6-3d-spike`（証跡として保持・**featureへはマージしない**）
**ゲート（roadmap §7C H6）**: 3D同梱ページのみ +150KB gzip 以内／他ページ影響ゼロ／`output:export` 両立。超過なら iso.ts の2.5Dへ切替（沈没コスト回避）。

## 実施内容
- `three@0.185` + `@react-three/fiber@9.6`（最小構成・drei無し）を導入。
- NCU地下BOXの最小3Dシーン `NcuScene3D.tsx`（蓋材質・アンテナ位置が入力に追従、低電力GL設定）。
- **ボタンゲート遅延読込**（`next/dynamic` ssr:false＋クリック時のみマウント。CollapsibleSection は閉時も子をマウントする仕様のため不使用）。
- 静的export・全ゲート（tsc/vitest 322/lint/build/e2e 35）緑。

## 計測結果
| 指標 | 値 | 判定 |
| --- | --- | --- |
| NCU初期ロード | 167→168 kB（**+1kB**） | ✅ ボタンゲートが有効 |
| 他ページ・shared | +1kB（102→103） | ✅ ほぼ影響なし |
| **クリック時に取得する three チャンク群** | **約226KB gzip**（98+84+44） | ❌ **予算150KBを76KB超過（1.5倍）** |
| `output:export` 両立 | ビルド成功 | ✅ |

## 追加で判明したコスト（重要）
- **R3F v9 のグローバルJSX型拡張が既存コンポーネントの型を破壊**: three要素が `ElementType` 全域に混入し、ポリモーフィックな `Card`（`as?: ElementType`）の children が `never` に潰れてビルド不能に。今回は Card の `as` を実使用タグの明示ユニオンに制約して解消したが、**three系を導入する限り、今後の全ポリモーフィック部品が同リスクを負う**。

## 判定
**予算超過 → 事前ゲートどおり 2.5D（等角SVG・`src/lib/ui/iso.ts` 実装済み）を正式経路として採用。**
- three/R3F の依存は feature に持ち込まない（本ブランチのみ）。
- Card の型強化（`as` 明示ユニオン化＝グローバル汚染への免疫）は**単独で価値がある**ため feature へ還元する。
- 3Dの将来再検討の条件: (a) アンテナ3D放射パターン等「2.5Dで表現不能な価値」が確定し、(b) 226KB gz のオンデマンド負担をユーザーが明示に許容する場合。その際も本スパイクのボタンゲート方式を踏襲。

## 2.5D経路の次アクション（H6改）
