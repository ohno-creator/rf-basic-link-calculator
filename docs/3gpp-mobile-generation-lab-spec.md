## 1. 企画の結論

このツールは「1Gから5Gまで何倍速くなったか」を並べる年表ではなく、次の問いに答える体験型アプリとする。

> **移動通信の世代が変わるたび、通信システムは何を新しく解決できるようになったのか。**

世代差の本質を、以下の3層を切り分けながら理解させる。

1. **無線アクセス（RAN）**: 限られた時間・周波数・符号・空間を、複数利用者へどう配るか。
2. **コアネットワーク**: 音声・データ・認証・移動・品質を、ネットワーク全体でどう扱うか。
3. **サービス要件**: 音声、メッセージ、Web、動画、IoT、産業制御など、何を保証したいか。

最重要の学習成果は、ユーザーが次の4点を自分の言葉で説明できることとする。

- 世代と周波数は別軸であり、「高い周波数だから5G」とは限らない。
- 世代は無線方式だけではなく、コア網とサービスを含むシステムの変化である。
- 5G NSAと5G SAは、同じ「5G表示」でも使用するコア網と提供可能な機能が異なる。
- 6Gは完成済み製品の性能表ではなく、標準化中の領域として扱う必要がある。

## 2. 既存アプリに追加する価値

### 2.1 既存ツールとの役割分担

既存の `cellular-band-map` は「周波数とBand番号」、`lte-signal-metrics` は「LTEの測定指標」を扱う。本ツールはその上位概念として、「なぜ世代ごとにBand、測定指標、基地局、コア網が変化したか」をつなぐ。

推奨学習導線:

1. `mobile-generation-lab` — 世代・RAN・コア網の全体像
2. `cellular-band-map` — 4G/5G Bandと周波数の関係
3. `lte-signal-metrics` — LTEのRSSI/RSRP/RSRQ
4. `free-space-loss` / `diffraction-shadow` — 世代名ではなく周波数で伝搬が変わることを確認
5. `rf-basic-link-calculator` — 実際のリンク成立条件へ接続

### 2.2 独自性

一般的な世代解説は、年代・最高速度・代表端末を表にして終わる。本ツールは以下で差別化する。

- 同じ街・同じ利用者数・同じ要求を、世代切替で比較する。
- 「無線」「コア網」「サービス」の3層を透過表示する。
- NSA/SAを配線のつなぎ替えとして体感できる。
- 世代と周波数を独立操作し、混同を意図的に崩す。
- 数値競争ではなく、各世代が解いた制約と残した制約を示す。

## 3. 対象ユーザーと利用場面

### 3.1 主対象

- 携帯通信のニュースで「5G」「SA」「ミリ波」「6G」を見ても関係が整理できない初学者
- RF設計は分かるが、移動通信システム全体の世代差を俯瞰したい技術者
- LTE-M / NB-IoT / ローカル5G端末を扱い、RANとコア網の境界を理解したい組込み開発者
- 高専・大学・企業研修で、移動通信の入口教材を必要とする講師・受講者

### 3.2 代表利用シナリオ

- 「5Gなのに4Gより遅いことがあるのはなぜ？」を3分で理解する。
- 「NSAとSAはアンテナ方式の違いか？」という誤解を解く。
- 4G端末の音声が回線交換ではなくIMS/VoLTEで成立する流れを確認する。
- 6Gの記事に出る目標値と、確定した標準仕様を区別する。

### 3.3 非対象

- 3GPP仕様書を置き換えるプロトコル実装リファレンス
- 実ネットワークのスループット予測器
- キャリア別エリア・料金・商用機能の比較
- 物理層波形、チャネル符号化、スケジューラの精密シミュレータ
- 1G〜5G全方式を網羅する歴史データベース

## 4. 用語と表示上の原則

### 4.1 混同してはいけない5軸

画面・データモデル・説明文では、以下を必ず別フィールドとして扱う。

| 軸 | 例 | 表示ルール |
|---|---|---|
| 世代 | 2G / 3G / 4G / 5G | 社会的・標準化上の大きな世代区分 |
| Release | Rel-8 / Rel-10 / Rel-15 / Rel-18 | 3GPP仕様セットの版。世代と1対1ではない |
| 無線アクセス方式 | GSM / UTRA / E-UTRA / NR | 端末と基地局の間の無線技術 |
| コアネットワーク | GSM core / packet core / EPC / 5GC | 認証・移動・セッション・ユーザーデータを扱う網 |
| 配備構成 | LTE / NSA / SA | 実際にどのRANとコアを組み合わせたか |

### 4.2 正確性のための必須注記

- **1G**: 3GPP成立以前の技術であり、「3GPPが1Gを規定した」と表示しない。
- **2G**: GSMは3GPP設立以前に始まり、後に3GPPの仕様体系へ継承された経緯を注記する。
- **3G**: 3GPPは1998年に、発展型GSMコアとUTRAを基礎とする3Gシステム仕様の策定を目的に設立された。
- **4G**: 初期LTEを一般的な4G呼称として扱いつつ、厳密にはLTE-Advanced（Rel-10）がIMT-Advanced要件を満たす節目であると注記する。
- **5G**: `5G = NR` と短絡させない。5G SystemはNR、NG-RAN、5GC、各種プロトコルとサービス能力を含む。
- **5G NSA**: NRを使用していても、LTE/EPCをアンカーとする構成がある。SAと同等の5GC機能を使えると表示しない。
- **5G-Advanced**: 独立した「6G」とせず、5GのRelease発展として表示する。
- **6G**: 2026年7月時点では標準化作業中として表示し、商用性能・確定仕様・確定周波数を断定しない。

### 4.3 速度値の扱い

世代ごとの「最大速度ランキング」を主表示にしない。速度を表示する場合は次を区別する。

- 規格上のピーク値
- 商用ネットワークでの代表的な実効値
- 個々の利用者が得る瞬時値

MVPでは出典・前提・帯域幅・アンテナ構成を伴わない速度数値を出さない。代わりに「音声中心」「低速データ」「モバイルブロードバンド」「多様なサービス要件」のような能力変化を主表示にする。

## 5. コンテンツモデル

### 5.1 世代カード

| 表示 | 中心課題 | 主な変化 | 残る制約 |
|---|---|---|---|
| 1G | 移動しながら音声通話 | アナログセルラー、セル間移動、周波数再利用 | 容量、秘話性、データ、地域ごとの方式分断 |
| 2G | 音声をデジタル化し加入者を増やす | デジタル音声、TDMA系、SMS、認証・暗号化、後期のパケットデータ | データ速度、回線交換中心、方式間の分断 |
| 3G | 携帯電話を本格的にデータ網へ接続 | UTRA/W-CDMA、パケットデータ、HSPA、音声とデータの併存 | 複雑な二重構造、動画時代の容量・効率 |
| 4G | モバイルブロードバンドを全IPで効率化 | LTE/E-UTRA、OFDMA系、EPC、IMS/VoLTE、MIMO、CA | 多用途保証、産業向け柔軟性、LTE/EPC依存 |
| 5G | 異なる通信要件を同一基盤で扱う | NR、柔軟な無線設計、5GC SBA、スライシング、エッジ連携、産業・IoT要件 | SA移行、複雑性、カバレッジと容量の両立 |
| 5G-A | 5Gを高度化し次世代へ橋渡し | Rel-18以降の高度化として整理 | 6Gとは区別し、Release単位で更新が必要 |
| 6G | 標準化中 | 「検討中」「研究中」「規定済み」を区別して表示 | 未確定事項を製品仕様のように扱えない |

この表は画面文言の骨格であり、方式の網羅表にはしない。各世代につき「解いた問題」「仕組み」「新しく可能になったこと」「次に残した問題」の4項目へ展開する。

### 5.2 3層断面データ

各世代は次の構造を持つ。

```ts
type MobileGenerationId = "1g" | "2g" | "3g" | "4g" | "5g" | "5g-advanced" | "6g";

type EvidenceState = "historical" | "specified" | "deployed" | "in-standardization" | "research-concept";

type GenerationProfile = {
  id: MobileGenerationId;
  label: string;
  eraLabel: string;
  evidenceState: EvidenceState;
  oneLineEssence: string;
  problemSolved: string;
  access: {
    technologies: string[];
    resourceMetaphor: string;
    factualSummary: string;
  };
  core: {
    technologies: string[];
    voicePath: string;
    dataPath: string;
    factualSummary: string;
  };
  services: {
    representative: string[];
    newlyPractical: string[];
  };
  limitations: string[];
  releaseMilestones: ReleaseMilestone[];
  caveats: string[];
  sources: SourceRef[];
};
```

`EvidenceState` は6Gだけでなく、歴史的事実・規格上の能力・商用配備を混ぜないため全世代で使用する。

### 5.3 Release表示

Releaseは世代スライダーとは別の補助タイムラインにする。MVPで表示する節目は必要最小限とする。

- 3G: 3GPP設立と初期3G仕様、HSPAへの発展
- 4G: Rel-8 LTE/EPC、Rel-10 LTE-Advanced
- 5G: Rel-15 5G Phase 1 / NR / 5GC、Rel-18 5G-Advanced開始
- 6G: 研究・Study・Normative workを確定状態ラベル付きで表示

Release番号を「AndroidやiOSのバージョン」のように世代置換と説明しない。同じ世代を継続的に拡張する仕様セットとして説明する。

#### 構造

```text

          無線アクセス          網内処理            利用目的
```

- 4Gでは音声もIMS/VoLTEとしてIP系を通ることを示す。
- 5GではNRだけでなく、5GCの機能分割とサービスベース化を示す。
- 略語を最初から大量表示せず、詳細を開いたときだけ正式名を出す。

### 6.5 電波資源の分け方ラボ

#### 目的

FDMA、TDMA、CDMA、OFDMA、空間多重を「方式の優劣」ではなく、限られた資源を利用者へ割り当てる異なる考え方として理解させる。

#### UI

- 横軸: 時間
- 縦軸: 周波数
- 色: 利用者A〜D
- オーバーレイ: 符号または空間レイヤー（必要な世代のみ）
- 利用者数: 1〜8のスライダー
- 表示モード: `概念図` のみ。実スケジューラ再現とは表示しない。

#### 世代ごとの概念表示

- 1G: 周波数チャネルを利用者ごとに占有する概念
- 2G: 周波数チャネル内の時間スロット分割の概念
- 3G: 同じ時間・周波数を符号で分ける概念
- 4G: 直交サブキャリアと時間単位の柔軟割当の概念
- 5G: OFDM系の柔軟性に加え、帯域・時間構造・空間利用を目的別に組み合わせる概念

#### 必須注意

- 世代には複数方式・複数Duplex構成があり、この図は代表的な考え方の比較である。
- 「3Gは常にCDMA」「4Gは下りも上りも同一方式」のような過剰一般化をしない。
- 色付き面積を、そのまま実効速度や周波数利用効率の定量値として読ませない。

### 6.6 5G NSA / SA切替

本ツールの主役となる体験とする。

#### 操作

- セグメント切替: `4G LTE` / `5G NSA` / `5G SA`
- 同じスマートフォン、LTE基地局、NR基地局、EPC、5GCを画面内に常駐させる。
- 切替時に使用中のノードと経路だけを強調する。

#### 表示すべき接続

- 4G LTE: UE → eNB → EPC → Service
- 5G NSA（代表例 EN-DC / Option 3系）: UE → LTE eNBをアンカー + NR側を追加 → EPC → Service
- 5G SA: UE → gNB → 5GC → Service

#### 主結果文

- LTE: `無線もコアも4G。モバイルブロードバンドを全IPで提供します。`
- NSA: `NRを使っていても、制御とコア網は4G側が土台です。早く5G無線を展開できる移行構成です。`
- SA: `NRが5G Coreへ直接つながり、5Gシステムの機能を前提に設計できます。`

#### 禁止表現

- NSAを「偽物の5G」と呼ばない。
- SAなら常にNSAより高速・低遅延と断定しない。
- 画面の5Gピクトだけからコア構成を特定できると説明しない。
- 5GCにつながれば、全商用網で全5G機能が必ず提供されると説明しない。

### 6.7 世代と周波数の分離実験

#### 目的

「5Gは高い周波数、4Gは低い周波数」という誤解を崩す。

#### 操作

- 世代: `4G LTE` / `5G NR`
- 周波数プリセット: `700MHz級` / `3.5GHz級` / `28GHz級`
- 比較条件: 同一送信電力・同一アンテナ利得・同一距離の概念比較

#### 表示

- 波長 `λ = c / f`
- 自由空間損失の周波数差 `ΔFSPL = 20 log10(f₂ / f₁)`
- 定性表示: 回折・建物透過・利用可能帯域幅・アンテナ寸法の傾向
- `cellular-band-map`、`free-space-loss`、`diffraction-shadow` へのリンク

#### RF上の前提

- FSPL差は同一距離・自由空間・同一アンテナ利得の比較と明記する。
- 実際のセル到達距離はEIRP、受信感度、帯域幅、MIMO/ビームフォーミング、基地局密度、地形・建物で変わる。
- 「5Gだから届かない」ではなく、「高周波・広帯域・配備条件の組合せでカバレッジ特性が変わる」と説明する。

### 6.8 誤解を解くチェック

5問を1問ずつ表示する。全問正解を目的にせず、回答直後に理由と該当図へのリンクを出す。

1. `5Gは必ず4Gより遠くへ届く` → 誤り
2. `5G表示なら必ず5G Coreを使用している` → 誤り
3. `28GHzだから5G、700MHzだから4Gである` → 誤り
4. `LTEの音声は従来の回線交換だけで運ばれる` → 誤り
5. `5Gの変化は最高速度だけではない` → 正しい

結果表示は点数だけで終わらせず、誤答の種類を次の3軸で返す。

- `周波数と世代の混同`
- `RANとCoreの混同`
- `規格能力と実配備の混同`

## 8. 情報設計と文章ルール

### 8.1 二層読者設計

- 表層: 1文の本質、概念図、操作結果
- 深層: 正式名称、Release、例外、出典、仕様番号

専門用語は「略語 → 日本語の役割 → 正式名」の順で開示する。

例:

> **EPC（4Gコア網）** — LTE端末の認証、移動、データ通信を管理する仕組み。正式名: Evolved Packet Core。

### 8.2 断定強度

表示文言には事実の状態を示すバッジを付けられる構造を持たせる。

- `歴史的事実`
- `3GPP規定`
- `商用配備例`
- `標準化中`
- `研究構想`

「規格で可能」と「日本の全キャリアで提供」を同じ文にしない。

### 8.3 比喩

比喩は理解の入口として使い、必ず破綻点を添える。

- FDMA: 車線を分ける
- TDMA: 同じ窓口を時間で交代する
- CDMA: 同じ部屋で異なる言語を使う
- OFDMA: 小さな時間×周波数の区画を必要に応じて配る
- SBA: 固定配線の専用装置群ではなく、許可された機能同士がサービスを呼び出す

注意: 実際の直交性、干渉、電力制御、スケジューリングを比喩だけで説明し切ったと見なさない。

## 9. ビジュアル仕様

### 9.1 基本方針

- 年表より「同じシーンが世代で変わる」比較を主役にする。
- 色は世代そのものより、3層の意味へ固定する。
  - Access: 青系
  - Core: 紫系
  - Service: 緑系
  - 注意・未確定: 黄〜橙系
- 1G〜6Gを虹色にして色だけで識別させない。
- 接続線は向き、線種、ラベルを併用する。

### 9.2 図の意味

- 実線: 主なユーザーデータ経路
- 破線: 制御・補助接続の概念
- 淡色: 構成要素は存在するが現在の比較では未使用
- 点滅は使わない。切替時の経路移動は最大300ms程度。

### 9.3 レスポンシブ

- Desktop: 左に操作、中央に図、右に結果・本質カード
- Tablet: 操作と結果を上、図を下
- Mobile: 操作 → 主結果 → 図 → 詳細の順。図内の文字を縮小しすぎず縦積みに変形する
- 360px幅で横スクロールを発生させない

## 10. アクセシビリティ

- 世代スライダーは `role="radiogroup"` または同等のネイティブ操作を使用する。
- NSA/SA切替は選択状態をスクリーンリーダーへ通知する。
- 経路図と資源割当図には、現在状態を文章で説明する代替テキスト領域を設ける。
- 色だけで利用者・層・状態を区別しない。記号、ラベル、線種を併用する。
- 動的結果は必要な範囲で `aria-live="polite"` を使用し、連続スライダー操作中の過剰読み上げを避ける。
- 全操作をキーボードで完結可能にする。
- `prefers-reduced-motion` に対応する。
- WCAG 2.2 AA相当のコントラストとフォーカス可視性を目標とする。

### 12.2 責務

- `mobileGenerations.ts`: 世代・Release・用語・出典の純データ。

- `MobileGenerationLabColumn.tsx`: 3GPP成立史、LTE/4G呼称、5G System、6G標準化状態の深掘り。

### 12.3 計算ロジック候補

MVPで純関数化する定量処理は、世代比較そのものではなく周波数差だけとする。

```text
λ[m] = 299,792,458 / f[Hz]

同一距離でのFSPL差[dB]
ΔFSPL = 20 log10(f2 / f1)
```

単位:

- 入力周波数: MHz
- 波長表示: m / mm
- 損失差: dB

前提:

- 自由空間
- 同一距離
- 同一送信電力
- 同一アンテナ利得

この比較は「世代による損失差」ではなく「周波数によるFSPL差」であることを関数名・UI・テスト名で明示する。

- 新規外部依存は追加しない。
- 既存のCard、Button、Input、Result、Source表示、チャートテーマを再利用する。
- Rechartsは資源グリッドに必須ではない。意味の単純な区画図はSVG/HTMLを優先する。
- サーバーAPI、Route Handler、Server Action、実行時外部取得を使用しない。

- 誤解チェックの回答後に根拠説明が表示される。
- モバイル幅で主操作と主結果が最初の流れに入る。

- Tablet: 768×1024

## 20. 最終的な成功指標

アクセス数だけではなく、以下の行動を成功とみなす。

- ユーザーがNSA/SA切替を操作し、RANとCoreの違いを見る。
- 世代と周波数を別々に切り替える。
- 世代ラボからBand地図、FSPL、LTE指標へ移動する。
- 誤解チェック後に、間違えた軸の説明へ戻る。

このツールの価値は「5Gの単語を覚えること」ではなく、以後の通信技術ニュースやRF設計判断を、**世代・Release・無線・コア・周波数・配備**に分解して読めるようにすることにある。
