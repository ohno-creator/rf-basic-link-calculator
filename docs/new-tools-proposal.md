# Track G: 新規基礎ツール20本 詳細仕様（正本）

**目的**: スタッフ株式会社のアンテナ事業（内蔵/外付けアンテナ製品・同軸ケーブルアセンブリ・アンテナ選定/筐体込み評価/OTA測定の相談）と相性の良い基礎ツール20本を追加する。
**位置づけ**: [improvement-roadmap.md](./improvement-roadmap.md) Track G の正本。既存24ツール・Track E（E1-E6）と重複しない。
**実装規約（全ツール共通）**:
- 進め方は Track E と同一: ① `src/lib/rf/<slug>.ts` 純関数＋期待値テスト（**テスト先行・Claude担当**）→ ② `src/data/tools.ts` に1エントリ → ③ ToolShell v2 テンプレート上にパネル（**Codex担当**）→ ④ e2eスモーク＋実機確認（**Antigravity**）。
- UIは ui-redesign-plan.md の標準（2ペイン・ResultBar・MetricCard・Field・プリセットチップ・折り畳み）に従う。**本書の各ツール仕様は「標準からの差分」だけを書く**。
- **[DATA]マークのツール**は係数・判定値を `src/data/` の出典付きテーブル（column-guide.md の `ColumnSource` 型）に分離する。**数値の発明禁止**——Antigravityの文献パック（一次出典）経由でのみ投入し、出典なしの値はマージ不可。
- エラーは RfError（コード）、文言は `rfErrorMessages.ts`。単位・適用条件は必ずUIに明記（AGENTS.md）。
- 新カテゴリ2つを `tools.ts` の `toolCategories` に追加: `implementation`（実装・筐体）/ `system`（方式・運用設計）。

**テスト期待値の扱い**: 本書の数値は仕様の一部（受け入れ基準）。実装者は独立に再計算して一致を確認し、不一致なら実装せず本書への質疑として差し戻すこと。

## グループ1: アンテナ実装・筐体設計（内蔵アンテナ事業の中核）

### G1 `ifa-initial-dimensions` — 逆F/IFAアンテナ初期寸法
- **事業フック**: 「内蔵アンテナは初期寸法どおりに動かない」体験→筐体込み評価・カスタム設計相談へ。外付け品（同軸接続）比較の導線。
- **中核式**: 全長目安 L ≈ (c/4f) / √εeff、εeff = (εr+1)/2（基板上パターンの簡易近似と明記）。短絡ピン〜給電間隔は L/8〜L/12 の目安レンジ表示。
- **入力**: 周波数MHz（既定920、min100、max6000）／基板εr（既定4.4）／基板厚mm（既定1.0）。プリセット: 920MHz/1575(GNSS)/2400(BLE)/LTE-M(800)。
- **出力**: primary=全長目安mm。secondary=自由空間λ/4・短縮率・短絡-給電間隔レンジ。
- **ビジュアル**: IFA上面図SVG（Lに連動して伸縮、既存 PatchAntenna 図の様式）。
- **警告（常時）**: 「初期値。筐体・GND・部品配置で±10〜20%ずれるため実測/EM解析での追い込みが前提」→ 相談CTA接続。
- **lib/テスト**: `calculateIfaDimensions`。**期待値: 920MHz・εr4.4 → λ/4=81.5mm、εeff=2.7、L≈49.6mm**。εr=1でL=λ/4に一致。
- 工数S／依存: UX-2テンプレート。

### G2 `ground-plane-size` — GNDプレーン寸法と効率 [DATA]
- **事業フック**: 「基板が小さくて性能が出ない」の定量化→GND非依存の外付けアンテナ提案・評価相談。
- **中核**: λ/4系アンテナのGND長 Lg/λ に対する (a)共振シフト傾向 (b)効率低下dB の**出典付き実測テーブル**（TI DN035級のアプリケーションノートから。Antigravity文献パック必須）。区分線形補間。
- **入力**: 周波数MHz／基板の最長GND辺mm。**出力**: primary=効率低下の目安dB（レンジ表示）。secondary=Lg/λ・推奨GND長（λ/4）。
- **ビジュアル**: Lg/λ vs 効率カーブ（現在点マーカー）。
- **lib/テスト**: テーブル補間の単調性・端点クランプ・**Lg=λ/4で低下0dB**。データ値自体の照合は出典レビュー（column-guide §5工程④）で担保。
- 工数M（データ整備が主）。

### G3 `antenna-keepout` — アンテナ・キープアウト領域チェック [DATA]
- **事業フック**: 「この基板サイズで内蔵できるか」の一次判定→不可なら外付け/ケーブル接続アンテナ（自社製品）へ誘導する自然な導線。**事業価値最上位**。
- **中核**: アンテナ種別（チップ/PCBパターン/フレキ/スプリング）×帯域→必要クリアランス（W×H mm）と GND禁止領域の**典型値テーブル**（主要メーカーデータシートの推奨実装から。出典必須）。ユーザーの確保可能領域と比較し 充足/不足/要相談 の3値判定。
- **入力**: 帯域プリセット（920/1575/2400/サブ6）／種別／確保可能な領域W×H。**出力**: primary=判定（success/caution/danger）。secondary=必要領域・不足量mm。
- **ビジュアル**: 基板外形とキープアウトの重ね描き（充足=緑/不足=赤）。
- **lib/テスト**: 判定境界（ちょうど充足/1mm不足）。
- 工数M。

### G4 `detuning-estimator` — 筐体・近接物による離調推定 [DATA]
- **事業フック**: 「ケースに入れたら飛ばなくなった」の説明装置→筐体込み評価サービスへ。
- **中核**: 近接シナリオ（樹脂カバー密着/1mm/3mm、手把持、金属近接）→共振シフト%とVSWR劣化の**典型レンジテーブル**（出典必須）。ユーザーのVSWR≤2帯域幅（G8と連携）と比較し「帯域から外れるか」を判定。
- **入力**: 中心周波数・VSWR≤2帯域幅MHz（不明なら比帯域プリセット）・シナリオ。**出力**: primary=帯域内に残るか判定。secondary=想定シフトMHzレンジ。
- **lib/テスト**: シフト%→MHz換算、判定境界。
- 工数M。

### G5 `body-loss` — 人体・手の影響（ボディロス） [DATA]
- **事業フック**: ウェアラブル/ハンドヘルド機器のリンク設計→リンクバジェット診断の環境損失欄へワンクリック転記（`environmentLossDb` への導線）。
- **中核**: 帯域×装着シナリオ（手持ち/胸ポケット/手首/頭部近接/体表固定）→追加損失dBの**文献レンジテーブル**（3GPP・査読論文。出典必須）。
- **入力**: 帯域・シナリオ。**出力**: primary=推奨追加損失（typicalとworst）。secondary=文献レンジ・根拠出典。
- **lib/テスト**: テーブル参照・レンジ整合（min≤typ≤max）。
- 工数S（データ整備が主）。

### G6 `metal-mount-effect` — 金属面近接・設置の影響
- **事業フック**: 「金属筐体/金属板に付けたい」相談の定番→オンメタル対応アンテナ・パッチ品の提案導線。
- **中核式**: 完全導体板に平行な素子の鏡像干渉。正面方向の電界係数 = 2·|sin(2πd/λ)|、利得変化 = 20log₁₀(2|sin(2πd/λ)|) dB。適用条件: 板がλ級以上・平行設置・理想反射（明記）。
- **入力**: 周波数MHz・金属面からの距離mm（スライダー）。**出力**: primary=利得変化dB。secondary=最適距離λ/4mm・ヌル条件。
- **ビジュアル**: d/λ vs 利得変化カーブ（d=λ/4で+6dB、d→0で急落）。
- **lib/テスト**: **d=λ/4→+6.02dB、d=λ/8→+3.01dB、d→0で−∞方向（クランプ−40dB表示）**。
- 工数S。

## グループ2: 整合・給電（ケーブルアセンブリ事業との相性）

### G7 `l-match` — L型整合回路計算
- **事業フック**: 整合検討の入口。「計算どおりに動かない」→実測チューニング・評価相談へ。
- **中核式**: 複素負荷 Z_L=R+jX を Z0=50Ω へ整合するL型2解（Bowick "RF Circuit Design" ch.4 の標準解法）。Q=√(max(R,Z0)/min(R,Z0) − 1)。素子値は nH/pF 換算。
- **入力**: R（既定20Ω）・X（既定−10Ω）・周波数MHz（既定920）・Z0（既定50、変更可）。
- **出力**: primary=解1（直列L＋並列C 等、素子値つき）。secondary=解2・回路Q・想定比帯域(≈1/Q)。
- **ビジュアル**: 回路図SVG2種（直列/並列素子のL/C記号が解に連動）。
- **警告**: R=Z0近傍は単一リアクタンスで整合（L型不要）と表示。R,X測定にはVNAが必要→測定サービス導線。
- **lib/テスト**: **Z_L=20−j10Ω・920MHz・50Ω → 解1: 直列L=5.97nH＋並列C=4.24pF／解2: 直列C=11.9pF＋並列L=7.06nH**（Q=1.225）。実装時にアドミタンス走査による独立検算をテストに含めること（2手法一致で合格）。
- 工数M。

### G8 `vswr-bandwidth-q` — VSWR帯域幅⇔Q⇔比帯域
- **事業フック**: データシートの「VSWR≤2帯域」を設計言語（Q）に翻訳→small-antenna-limit（Chu限界）と直結し「その筐体では物理的に無理」を示せる。
- **中核式**: FBW(VSWR≤s) = (s−1)/(Q·√s)（単共振近似。Yaghjian & Best 2005）。双方向換算（FBW→Q も）。
- **入力**: モード切替（Q既知/帯域既知）＋ s（既定2、選択1.5/2/3）。**出力**: primary=比帯域% or Q。secondary=中心周波数指定時の帯域MHz・Chu限界との比較リンク。
- **lib/テスト**: **Q=20・s=2 → FBW=3.54%**。逆変換の往復一致。
- 工数S。

### G9 `mismatch-range-impact` — ミスマッチ損失→通信距離影響
- **事業フック**: 「VSWR 3ってどのくらい悪いの？」への即答→整合改善・アンテナ交換の価値を距離%で示す営業ツール。**工数最小・即効**。
- **中核式**: 既存 `vswr.ts` の ML[dB] → 自由空間の距離倍率 = 10^(−ML/20)（既存 `db.ts` の dbToDistanceRatio 再利用）。
- **入力**: VSWR（スライダー1〜10、既定3）。**出力**: primary=距離影響%。secondary=ML dB・反射電力%。
- **ビジュアル**: VSWR vs 距離% カーブ＋現在点。
- **lib/テスト**: **VSWR=3 → Γ=0.5、ML=1.25dB、距離−13.4%**。VSWR=1→0%。新libほぼ不要（組合せ関数のみ）。
- 工数S（最小）。

### G10 `cable-phase-length` — ケーブル電気長・位相計算
- **事業フック**: ケーブルアセンブリ事業と直結。位相指定品・長さ公差の問い合わせを定量化（アレイ/MIMO/ダイバーシティ給電の位相合わせ）。
- **中核式**: 電気長[λ] = L/(VF·λ0)、位相 = 360°·L/(VF·λ0)、長さ差ΔL→位相差Δφ、許容位相差→必要長さ公差。
- **入力**: 周波数MHz・物理長mm・VF（既定0.66、ケーブル品番プリセットで連動: 既存 `coaxCables.ts` を再利用）・比較モードでΔL。
- **出力**: primary=位相角°。secondary=電気長λ・1mmあたり位相・公差換算。
- **lib/テスト**: **920MHz・VF0.66 → ケーブル内波長215.1mm、1mm=1.674°**。1波長=VF·λ0。
- 工数S。

## グループ3: 複数アンテナ・ダイバーシティ（MIMO/複数アンテナ製品）

### G11 `antenna-isolation` — 2アンテナ間結合量の目安
- **事業フック**: MIMO/ダイバーシティ実装の間隔設計→2アンテナセット品・評価サービス。既存 antenna-spacing（λ換算）の「で、何dB結合するの？」に答える続編。
- **中核式**: 遠方界近似 S21[dB] ≈ 20log₁₀(λ/(4πd)) + G1 + G2。適用条件: d≳λ/2・偏波平行・自由空間（**共有GND上の実装ではこれより悪化**と常時明記）。
- **入力**: 周波数・間隔mm・各利得dBi（既定2.15）。**出力**: primary=結合量dB＋判定（≤−15dB良好/−10〜−15注意/>−10不足）。secondary=推奨間隔（−15dB達成距離）。
- **lib/テスト**: **d=λ/2・G=2.15dBi×2 → S21≈−11.7dB**。d<λ/2で「近傍界のため参考値」警告フラグ。
- 工数S。

### G12 `diversity-gain` — ダイバーシティ利得推定 [DATA(出典)]
- **事業フック**: 「アンテナ2本にする価値」の定量化→2本目のアンテナ販売・配置相談。
- **中核式**: 2ブランチ選択合成・レイリー環境。独立時: P_out=(1−e^(−x))^N を解き、**1%アウテージで利得≈10.2dB**（教科書値）。相関補正は係数√(1−ρe)（Vaughan & Andersen 1987 を出典に、近似と明記）。ρe は間隔から Clarke モデル ρe≈J0²(2πd/λ) で推定（一様到来の仮定を明記）。
- **入力**: 間隔（→ρe自動計算・手入力も可）・アウテージ率(1/5/10%)。**出力**: primary=ダイバーシティ利得dB。secondary=ρe・「ρe<0.5で効果十分」判定。
- **lib/テスト**: **ρe=0・1% → 10.2dB**（単独−19.98dB vs 選択−9.77dB）。ρe=0.7 → ×√0.3≈0.548倍。J0²(2πd/λ): d=0.5λ→ρe≈0.09。
- 工数M。

### G13 `pointing-margin` — ビーム幅と設置角度マージン
- **事業フック**: 指向性アンテナ（パッチ/八木/パラボラ）販売時の「どのくらい正確に向ければいいか」に即答。aperture-gain-beamwidth の続編。
- **中核式**: 放物線近似 L(θ)=12·(θ/HPBW)² dB（3GPP TR 36.814 アンテナモデル。HPBWは全幅）。逆算: 許容損失LdBの角度誤差 θ=HPBW·√(L/12)。
- **入力**: HPBW°（aperture-gainツールから遷移時は引継ぎ）・許容損失dB（既定1）。**出力**: primary=許容誤差角±°。secondary=3dB/1dB/0.5dB各マージン表。
- **lib/テスト**: **HPBW=30°・1dB → ±8.66°**。θ=HPBW/2で3dB。適用条件: メインローブ内（θ<HPBW程度）。
- 工数S。

## グループ4: 電波環境・現場（NCU/フィールド系顧客の横展開）

### G14 `wall-penetration` — 壁・建材の透過損失バジェット [DATA]
- **事業フック**: 屋内IoT（スマートメーター/ビル管理）の定番質問。nami-gate（窓）の壁版。リンクバジェット診断の環境損失へ転記導線。
- **中核**: 建材×周波数の透過損失テーブル（ITU-R P.2040 / NIST TN 6055 系の公開データ。出典必須）。複数壁の加算＋「合計をリンクバジェットへ」ボタン。
- **入力**: 周波数帯・壁リスト（材質×枚数を ChoiceChips＋カウンタで）。**出力**: primary=合計透過損失dB（レンジ）。secondary=内訳表。
- **lib/テスト**: 加算の線形性・テーブル端点。データは出典レビューで担保。
- 工数M。

### G15 `signal-metrics-converter` — RSSI/RSRP/RSRQ/SINR 変換・良否判定 [DATA(判定基準)]
- **事業フック**: LTE-M/NB-IoT 現場のトップ質問「この RSRP は良いのか」。NCUツールの現場分析と相互リンク。
- **中核式**: RSRP ≈ RSSI − 10log₁₀(12·N_RB)（全RB負荷近似と明記）。N_RB: 1.4MHz=6/3=15/5=25/10=50/15=75/20=100。RSRQ = N_RB·RSRP/RSSI（dB式で表示）。判定バンド（例: RSRP −90/−105/−115境界）は**キャリア/3GPP資料の出典付きデータ**とする。
- **入力**: 帯域幅・分かっている指標1つ。**出力**: primary=換算値＋良否バッジ。secondary=他指標・判定根拠。
- **lib/テスト**: **RSSI−70dBm・10MHz(50RB) → RSRP≈−97.8dBm**（補正27.78dB）。
- 工数S。

### G16 `desense-calculator` — 自家中毒（デセンス）と距離影響
- **事業フック**: 「基板ノイズでアンテナ性能が死ぬ」IoT頻出問題の定量化→OTA測定・ノイズ対策評価サービスへの最短導線。**技術的差別化が大きい**。
- **中核式**: 合成雑音床 N' = 10log₁₀(10^(N/10)+10^(I/10))。感度劣化Δ=N'−N。距離影響 = 10^(−Δ/(10n))（n=2自由空間、n可変）。E1（雑音床ツール）と入力を共有。
- **入力**: 基準雑音床 or (BW+NF)・干渉レベルdBm（アンテナ端）・距離指数n。**出力**: primary=感度劣化dB。secondary=距離影響%・合成雑音床。
- **ビジュアル**: 干渉レベル vs 距離影響カーブ。
- **lib/テスト**: **I=N → +3.01dB劣化 → 自由空間で距離−29.3%**。I=N−10 → +0.41dB。
- 工数S。

### G17 `measurement-sampling` — 電測の測定点数・ばらつき計画
- **事業フック**: 「実測しましょう」の次の質問「何点測ればいい？」に答える。研究距離シート・Track D6（実測方法論）と連動し、測定サービスの信頼性を裏付け。
- **中核式**: 必要点数 n=(z·σ/E)²（正規近似。z: 90%=1.645/95%=1.96）。ローカル平均の窓長は40λ（Lee 1985 を出典）。
- **入力**: シャドウイングσdB（プリセット都市8/郊外6/開放4）・許容誤差±EdB・信頼水準。**出力**: primary=必要測定点数。secondary=移動平均窓の長さm（40λ）・測定グリッド目安。
- **lib/テスト**: **σ=8dB・±2dB・95% → n=62点**（61.5切上げ）。920MHzの40λ=13.0m。
- 工数S。

## グループ5: 方式・運用設計（アンテナ選定ナビへの導線）

### G18 `lora-airtime` — LoRa SF・エアタイム・920MHz送信制限チェック [DATA(規制値)]
- **事業フック**: 920MHz LPWA顧客の設計中核。E4（EIRP法規）と対で「電波を出してよい条件」を完結させ、920MHz帯アンテナ製品へ誘導。
- **中核式**: Semtech AN1200.13 のシンボル数式で ToA を計算（T_sym=2^SF/BW）。ARIB STD-T108 の送信時間制限（キャリアセンス区分・休止時間・時間総量）は**規格の出典付きデータ層**に分離し照合。
- **入力**: SF(7-12)・BW(125/250/500kHz)・ペイロード長・CR・ヘッダ/CRC。**出力**: primary=エアタイムms＋規制適合バッジ。secondary=1時間あたり送信可能回数・SF別比較表。
- **lib/テスト**: 期待値ベクトル3点を**Semtech公式計算機から取得して出典として記録**し固定（実装値と±0.1ms一致）。SF+1でToA約2倍の性質テスト。
- 工数M。

### G19 `battery-life` — 電池寿命×送信条件トレードオフ
- **事業フック**: 「アンテナ利得を上げる＝送信電力を下げられる＝電池が持つ」をつなぐ、高利得アンテナの価値説明ツール。G18・E1と連携。
- **中核式**: 平均電流 = (I_tx·t_tx + I_rx·t_rx)/周期 + I_sleep。寿命h = 容量mAh·derate / 平均電流mA。derate（自己放電・温度）既定0.7、10年超は「電池自己放電が支配的」注記。
- **入力**: 容量mAh（既定2400）・送信電流mA/時間ms・送信間隔・スリープ電流µA・derate。プリセット: LoRa毎時/LTE-M毎日など。
- **出力**: primary=推定寿命（年）。secondary=平均電流µA・内訳円グラフ・「送信電力−3dB時」の比較行（G9と逆方向の連携）。
- **lib/テスト**: **2400mAh・TX45mA×50ms/h・sleep5µA・derate1.0 → 平均5.625µA・48.7年**（表示は「>10年（電池特性が支配）」）。derate0.7の乗算。
- 工数S。

### G20 `gnss-cn0` — GNSS受信 C/N0 バジェット
- **事業フック**: GNSSアンテナ（パッチ/アクティブ）の選定ツール。「パッシブで足りるか、アクティブが要るか」を即断→GNSS製品ラインへ誘導。
- **中核式**: C/N0[dB-Hz] = P_r + G_ant − L_preLNA − (−174 + NF_sys)。アクティブアンテナは Friis 縦続で NF_sys ≈ NF_LNA + (後段寄与/G_LNA)（後段ケーブル損失が実質無効化されることを可視化）。
- **入力**: 受信信号レベル（既定−130dBm/GPS L1オープンスカイ・出典明記）・アンテナ利得・LNA有無（利得/NF）・ケーブル損失・受信機NF。
- **出力**: primary=C/N0 dB-Hz＋判定（>40良好/35-40可/<35厳しい、受信機一般値として出典付き）。secondary=パッシブvsアクティブ比較の2列表示。
- **lib/テスト**: **P=−130dBm・G=3dBi・NF_sys=1.5dB・損失0 → C/N0=45.5dB-Hz**。ケーブル3dB: パッシブは−3、アクティブ(G=20dB)は−0.1程度に留まる縦続計算。
- 工数M。
