## プロジェクト概要

- **名称**: RF Basic Link Calculator（アンテナ・無線 基礎計算ツール群）
- **フレームワーク**: Next.js 15（App Router）
- **言語**: TypeScript
- **スタイル**: Tailwind CSS
- **グラフ**: Recharts
- **ホスティング**: GitHub Pages
- **ビルド形態**: `output: "export"`（静的書き出し）

### 構成の要点

- `src/lib/rf/*` … RF計算の純関数（テスト対象の中心）
- `src/data/*` … プリセット・辞書・ツールメタ（純データ）
- `src/app/tools/<slug>/` … 各ツールの画面（`page.tsx` は薄く保つ）
- `src/components/*` … 汎用UI
- `src/tests/*` … `lib/rf` に対応する単体テスト（Vitest）
- `e2e/` … Playwright

## RF計算ルール

RF計算を扱う際は、以下を必ず守る。

- **dB（対数）と線形値を混同しない**。加算はdB領域、乗算・比は線形領域で扱う。
- **単位を必ず明記**する（dB / dBm / dBi / dBW / W / mW / Hz / MHz / GHz / m / km など）。
- 主要な計算・モデルは適用条件を確認してから使う。
  - **Friis 伝達公式**（自由空間・遠方界前提）
  - **FSPL（自由空間伝搬損失）**
  - **Link Budget（リンクバジェット）**
  - **EIRP（等価等方輻射電力）**
  - **VSWR（電圧定在波比）**
  - **Return Loss（反射損失）**
  - **Reflection Coefficient（反射係数 Γ）**
  - **Noise Figure（雑音指数）**
  - **Thermal Noise（熱雑音, kTB）**
  - **Fresnel Zone（フレネルゾーン）**
  - **Two-ray（2波モデル）**
- **適用条件を確認**する（遠方界か、周波数・距離・アンテナ高の妥当範囲か、モデルの前提が成立するか）。前提を外れる入力では警告・注記を添える。
- 数式・定数の根拠（32.44 など FSPL 定数の単位系）を明示する。

## レビュー方針

指摘は **重要度（P0〜P3）** と **確度** の2軸で示す。

### 重要度

- **P0**: 計算結果が誤る／クラッシュする／データ破壊など、即修正が必要な致命的問題。
- **P1**: 実害があるが回避可能、または限定条件で誤る問題。早期修正推奨。
- **P2**: 品質・保守性・軽微な不整合。計画的に改善。
- **P3**: 好み・軽微な提案。任意対応。

### 確度

- **確定**: 再現・根拠が明確で断定できる。
- **高**: 強く疑われるが最終確認が望ましい。
- **要確認**: 仮説段階。裏取りが必要。

指摘は「重要度 + 確度 + 対象 `file:line` + 根拠 + 修正案」をセットで書く。
