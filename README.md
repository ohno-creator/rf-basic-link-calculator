# RF Basic Link Calculator

スタッフ株式会社向けの「通信距離・リンクバジェット簡易診断」Webアプリです。周波数、距離、送信電力、アンテナ利得、受信感度から、無線通信の受信電力とリンクマージンを簡易計算します。

## 主な機能

- 周波数・波長計算
- dBm / mW / W 変換
- 自由空間損失 FSPL 計算
- リンクバジェット簡易診断
- 通信形態選択（携帯基地局 → IoT端末、プライベート基地局 → IoT端末、ゲートウェイ → 低高度端末、低高度端末 ↔ 低高度端末、カスタム）
- 伝搬モデル選択（自由空間損失、2波、Log-distance、実測補正、IoT実測補正Hata、奥村-秦、COST 231-Hata）
- 送信側・受信側の空中線地上高入力（奥村-秦/COST 231-Hataでは hb/hm として伝搬損失に反映）
- 奥村-秦/COST 231-Hataのエリア種別入力（市街地、郊外、開放地の補正を切替）
- IoT実測補正Hataモード（既知距離のRSSI/RSRPからHata系モデルのオフセットと距離勾配を校正）
- GL以下NCU・水道BOX診断（水道BOX、メーターボックス、マンホール、地下ピット内端末を、地上側伝搬と蓋・BOX・水分・アンテナ位置による追加損失レンジに分けて評価。現場RSSI/RSRP差分から、蓋、BOX内配置、水分、車両遮蔽、反射・近傍フェージング、計算値とのズレを原因ランキング化。RSRP/RSSI/RSRQ/SINR/SNR/パケット成功率/再送回数の任意入力による通信品質指標の簡易診断にも対応）
- Log-distanceモデルの距離損失指数入力
- 端末近傍損失（地面近接、筐体、偏波ミスマッチ、車両・人体遮蔽、設置ばらつき）と実測補正値
- 伝搬モデルの適用範囲チェック、2波モデルのブレークポイント確認、日本語警告バナー
- 920MHz LPWA、BLE、Wi-Fi、LTE-M、金属筐体の悪い例プリセット
- はじめての見方ロードマップ、入力影響ガイド、モデル前提条件ガイド、結果読み取りガイド
- Signal Flow Diagram、Link Margin Gauge、Distance vs Received Power Chart
- Link Budget Waterfall Chart による利得・損失の可視化と、計算前提を示す2D断面図
- 2波モデル実験室（直接波＋地面反射波の干渉で、伝搬損失グラフが波打つ様子を可視化）
- 研究ベース距離計算シート（CIモデル、Dual-slope CI、IEEE 802.16 SUI Terrain A/B/C、COST231 Walfisch-Ikegami NLOS、3GPP TR 38.901 UMi/UMa LOS/NLOS、信頼率つき最大距離逆算）
- シャドウフェージング標準偏差、目標信頼率、追加フェード余裕を使った現実寄りの距離見積もり
- Sensitivity Line Visual、Wavelength Visual、Decibel Scale Visual
- 改善シミュレーションと相談用テキストコピー
- 入力条件の共有リンク生成・URL/ローカル保存・初期値リセット
- スクロール追従の判定サマリ（モバイル）
- RF学習クエスト（入門200問＋初心者・見習い・実務者・玄人・研究者の5モード×100問＝700問。選択肢ランダム表示、問題→即答え→解説→関連ツール→現場コラムで学習。10問ごとの章表示、レベルアップ表示、次の問題へボタン、ブラウザ進捗保存つき。各モード全問クリア後はランダム10問の修了試験を受け、100点で氏名・会社名入りのPDF修了書を出力可能）
- 基本計算ツール集（単機能・専用ページ）：VSWR・リターンロス変換、同軸ケーブル損失、マイクロストリップ線路（特性インピーダンス・電気長・マイター曲げ・ビアピッチ）、フレネルゾーン半径、伝搬損失（奥村-秦／COST 231-Hata）。各ページに入力連動の動的図と本質解説を搭載
- FAQ、SEO metadata、JSON-LD、コラム/お問い合わせ導線

## UI/UX方針

無線・アンテナ・dBに慣れていないユーザーでも、ロードマップで見る順番を把握し、プリセットを選び、入力影響ガイドを見ながらスライダーを動かし、滝グラフと結果読み取りガイドでリンクバジェットを理解できる構成です。BtoB製造業サイトに掲載できるよう、清潔感、実務性、技術的な信頼感を重視しています。

入力した条件はブラウザのlocalStorageに自動保存され、URLクエリにも反映されます。「条件を共有リンクでコピー」で得たURLを開くと同じ条件が復元され、自動的に診断結果までスクロールします。条件が初期値のときはURL・保存をクリーンに保ち、「初期値に戻す」でいつでもリセットできます。シリアライズ処理（`src/lib/rf/share.ts`）はURL・保存値を信頼境界の外側として扱い、不正値は既定値にフォールバックします。

本ツールは、マクロセル向けの経験式だけで通信可否を判定するものではありません。特に低高度IoT通信では、地面反射、フレネルゾーン欠損、筐体損失、車両・人体遮蔽、設置ばらつきの影響が大きくなります。そのため、本ツールではリンクバジェット、伝搬モデル、端末近傍損失、環境損失、実測補正を組み合わせて一次評価を行います。最終的な通信可否は、現地でのRSSIまたはRSRP測定により確認してください。

奥村-秦モデルおよびCOST 231-Hataモデルは削除せず、高所基地局と移動局間の広域平均伝搬損失を評価する参考モデルとして残しています。ただし、低高度端末同士の通信では主モデルとして推奨せず、自由空間損失、2波モデル、Log-distanceモデル、実測補正モデルを中心に評価する設計です。

奥村-秦/COST 231-Hataで用いる空中線地上高は固定値ではありません。リンクバジェット診断では、送信側アンテナ高を基地局高 `hb`、受信側アンテナ高を移動局高 `hm` として入力値から計算します。画面上でもHata系モデルを選択した場合に、現在の `hb` / `hm` と一般的な適用目安（基地局高30〜200m、移動局高1〜10m、距離1〜20km）を明示します。また、エリア種別も固定せず、市街地（大都市/中小都市）、郊外、開放地を選択できるようにし、Hataのエリア補正に反映します。

IoT実測補正Hataモードは、奥村-秦/COST 231-Hataを「現地実測で校正する基準線」として扱います。近年のLoRaWAN/NB-IoT測定研究では、固定の経験式だけでなく、現地のRSSI/RSRP、構造物、環境特徴量、距離勾配を取り込むことで誤差が下がることが示されています。本ツールでは、既知距離で取得したRSSIまたはRSRPからHata系モデルのオフセットを推定し、必要に応じて距離10倍あたりの勾配補正を加えます。単一点補正はアンカー距離近傍の補正であり、広い距離範囲では複数地点の実測で勾配を確認してください。

入力画面では、伝搬モデルごとに「使う入力」「前提」「現実とずれやすい点」を確認できるガイドを表示します。各入力ラベル横の `?` でマウスオーバー説明を確認でき、入力前提チェックメニューでは通信形態、推奨モデル、距離・高さ、損失の入れ分けを折りたたみで確認できます。特に、環境損失、端末近傍損失、実測補正値は役割が重なりやすいため、同じ筐体損失・遮蔽損失・測定差分を複数欄に入れないよう、二重計上の注意を明示しています。IoT実測補正Hataモードではアンカー実測点から基準モデルとの差分を推定するため、通常の実測補正値は別要因の追加補正だけに使う設計です。

2波モデルは、直接波と地面反射波の単純化モデルです。リンクバジェット判定では一点の干渉谷を過信しないよう平滑化した包絡線を使いますが、図解タブと伝搬損失モデル比較ページでは、直接波と反射波を位相込みで合成した完全版も表示し、強め合い・弱め合いでグラフが波打つ様子を確認できます。通信距離がブレークポイント付近または内側の場合は、設置位置や高さの小さな変化でRSSI/RSRPが大きく変わるため、実測確認を促します。Log-distanceモデルでは距離損失指数 `n` を入力値として扱い、初期値は一次評価用の仮定値です。実環境ではRSSIまたはRSRP測定に合わせて `n` と実測補正値を調整してください。

研究ベース距離計算シートでは、最新の3GPP TR 38.901 Release 19系で重視されるLOS/NLOS、アンテナ高、シャドウフェージング、7〜24GHz帯のモデル補正議論、および実測フィットで使いやすいCIモデルの考え方を、一次評価用の最大距離逆算として実装しています。加えて、IEEE 802.16系で使われたSUI Terrain A/B/C、都市街路の屋根越し・街路回折を扱うCOST231 Walfisch-Ikegami NLOSも比較モデルとして追加しています。平均伝搬損失が受信感度を上回るかだけではなく、目標信頼率に応じて `z × σ` のシャドウフェージング余裕と追加フェード余裕を差し引き、クラッタ・環境損失、端末近傍損失、実測補正値を含めた許容中央値伝搬損失から最大距離を求めます。

米国・日本の携帯キャリアの基地局設計では、標準式だけでなく、地図クラッタ、建物高、道路幅、アンテナパターン、干渉、トラフィック、ドライブテストやRSRP測定による補正を組み合わせます。本ツールは詳細なレイトレースやセル容量設計までは行いませんが、標準・IEEE系・都市街路系・実測補正を比較できるようにし、どの前提で距離が伸びたり縮んだりするかを把握しやすくしています。

2026年時点の研究動向として、IoT向けに奥村-秦モデルを万能に置き換える単一式が確立しているというより、実測データで既存モデルを校正し、構造物・環境特徴量・残差分布・信頼率マージンを組み合わせる方向が強くなっています。そのため本ツールでは、Hata系モデルを参考基準として残しつつ、IoT実測補正Hata、Log-distance、研究ベース距離計算、端末近傍損失、実測補正値を併用できる構成にしています。

GL以下NCU・水道BOX診断では、地面より下にある端末を奥村-秦モデルや2波モデルのアンテナ高へマイナス値として入れず、地上側伝搬損失と、蓋・BOX・GL下深さ・水分・アンテナ位置・開口・車両遮蔽による追加損失を分けて評価します。地下・水道BOX・マンホール内端末は、距離や高さだけで決まるのではなく、蓋材質、金属枠、湿潤、開口、アンテナ配置、現地構造物の影響が大きいため、結果は楽観・標準・厳しめのレンジで表示します。また、現場解析モードでは、BOX外、蓋開け、蓋閉め、雨天後、アンテナ改善後、車両あり、30cm移動/向き変更のRSSI/RSRPを入力し、差分から原因候補をランキングします。加えて、RSRP/RSSI/RSRQ/SINR/SNR、パケット成功率、再送回数を分かる範囲で入力し、受信電力不足、品質・干渉劣化、実通信成功率低下のどれが主因候補かを簡易診断します。計算値との差分は実測補正値として反映できるため、事前見積もりと現場原因追い込みをつなげて使えます。

RF学習クエストの研究者モードでは、2025〜2026年のLoRaWAN/IoT伝搬研究を題材に、環境特徴量、RSSI平滑化、非ガウス残差、フェードマージン、AERPAWでの高度・移動体比較、ITU-R P.1812-7の地理データ入力、3GPP Release 19のTR 38.901拡張論点を、100問の出典リンクつき問題とコラムとして学べるようにしています。後半50問では、世界の無線コミュニティで頻出するSWR、アンテナチューナ、dBi/dBd、グランドプレーン、コモンモード、LoRaのRSSI/SNRやSFの誤解も、一次資料や研究資料へつながる形で扱います。

入門モードでは、RF、アンテナ、伝搬、セルラー/LPWA、測定、品質評価、法規・認証、研究データで最初に知っておきたい200語を扱います。スタッフ株式会社の情報メディアにあるアンテナ基礎、放射効率、VSWR、ケーブル付きアンテナ設置、IP67防水ホイップ、平面型反射板の記事も参照し、製品実装・設置の入口として使える問題を組み込んでいます。難しい用語は問題文内に「用語メモ」を入れ、選択肢は混同しやすい近い概念も含めて、単なる暗記ではなく会話で使える理解につながるようにしています。各モードを全問クリアすると、そのモード内からランダムに10問を出題する修了試験が解放されます。修了試験は100点で合格とし、合格後に氏名と会社名を入力すると、ブラウザの印刷機能からPDF保存できる修了書を出力できます。通常問題・修了試験ともに、誤答した場合は対象問題を再クリック、または再挑戦ボタンで回答をクリアして再回答できます。

## 図解コンポーネント

- `SignalFlowDiagram`: 送信電力、アンテナ利得、損失、受信電力、リンクマージンの流れをカードで表示
- `BeginnerRoadmap`: 初心者向けにプリセット、入力、滝グラフ、実機確認の順番を表示
- `InputImpactGuide`: 距離、利得、損失、受信感度が結果にどう効くかを表示
- `ModelAssumptionGuide`: 各伝搬モデルが使う入力、前提条件、現実とのずれ、損失の二重計上リスクを表示
- `ResultReadingGuide`: 判定、主要3数値、滝グラフの読む順番を表示
- `LinkBudgetWaterfallChart`: 送信電力から利得・損失を積み上げ、受信電力に落ちるまでを滝グラフで表示
- `LinkAssumptionDiagram`: 滝グラフ直下に、送信側・受信側・2D距離・アンテナ高・見通し線・地面反射・指定パラメータ・注意点を表示
- `RadioPathDiagram`: 送信側、空間損失、ケーブル・筐体・環境、受信側を経路図として表示
- `LinkMarginGauge`: 0dB、10dB、20dBの判定基準をゲージで表示
- `DistancePowerChart`: 距離を変えたときの受信電力と受信感度ラインを表示
- `ResearchDistanceSheet`: 研究・標準モデルを使い、目標信頼率つきの最大通信距離を別シートで逆算
- `TwoRayInterferenceLab`: 直接波と地面反射波のコヒーレント和を表示し、2波モデルの山谷と平滑化線を比較
- `SensitivityLineVisual`: 受信電力と受信感度の上下関係を視覚化
- `WavelengthVisual`: 半波長λ/2（アンテナの基準寸法）を表示し、共振の本質・誘電率などによる小型化手法・小型化に伴う特性劣化を解説
- `DecibelScaleVisual`: dBmとmW/Wの関係、+10dBで10倍の感覚を表示

## 計算式

```text
λ[m] = 299,792,458 / 周波数[Hz]
```

```text
mW = 10 ^ (dBm / 10)
dBm = 10 × log10(mW)
W = mW / 1000
```

```text
FSPL[dB] = 32.44 + 20log10(距離[km]) + 20log10(周波数[MHz])
```

```text
受信電力[dBm]
= 送信電力[dBm]
+ 送信アンテナ利得[dBi]
+ 受信アンテナ利得[dBi]
- 伝搬損失[dB]
- ケーブル・コネクタ損失[dB]
- 環境損失[dB]
- 端末近傍損失[dB]
+ 実測補正値[dB]

リンクマージン[dB]
= 受信電力[dBm] - 受信感度[dBm]
```

```text
端末近傍損失[dB]
= 地面近接損失[dB]
+ 筐体損失[dB]
+ 偏波ミスマッチ損失[dB]
+ 車両・人体遮蔽損失[dB]
+ 設置ばらつきマージン[dB]
```

```text
Log-distance伝搬損失[dB]
= 1m基準の自由空間損失[dB] + 10 × 距離損失指数n × log10(距離[m] / 1m)
```

```text
2波モデル ブレークポイント[m] = 4 × 送信側アンテナ高[m] × 受信側アンテナ高[m] / 波長[m]

完全2波モデル（図解用）:
直接波経路長 r1 = √(d² + (ht−hr)²)
反射波経路長 r2 = √(d² + (ht+hr)²)
位相差 φ = 2π × (r2−r1) / λ
合成振幅² = 1 + Γ² + 2Γcos(φ)
```

### IoT実測補正Hataの根拠

- Urban LoRaの大規模測定研究では、112,000件超の受信電力サンプルで11種類の伝搬モデルを比較し、Okumura系やLog-distance系が都市環境で有効な候補である一方、現地データに基づく係数推定が重要であることが示されています。
- 2025年の屋内LoRaWAN測定データ研究では、壁などの構造物に加えて温湿度、CO2、粒子状物質などの環境特徴量を入れたモデルでRMSEが10.58dBから8.04dBへ改善しています。
- 2025年の環境認識型LoRaWAN研究では、環境特徴量とSNRを入れた統計モデルでRMSEが8.07dBから7.09dBへ改善し、残差分布から信頼率に応じたフェードマージンを校正しています。
- 2026年のAERPAW LoRaWAN実測研究では、地上車両、UAV、ヘリカイトを比較し、高度、移動、地形、NLOS条件が受信品質とモデルばらつきに大きく効くことが示されています。
- NB-IoTの深部屋内測定研究では、既存の経験式だけでは2〜12dB程度誤差が増える場合があり、距離以外の特徴量が必要であることが示されています。
- ITU-R P.1812-7を用いた2025年の農村部推定研究でも、標高・土地被覆・クラッタ高などの地理データが推定精度に影響することが示されています。
