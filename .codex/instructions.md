## Codex の役割

このプロジェクトにおける Codex の主担当は **実装（コードの追加・修正）**です。
設計レビューと影響範囲の見極めは Claude Code が主担当（`CLAUDE.md`）とし、Codex は合意済みの方針を **最小差分で正確に実装**します。

---

## 実装ルール

- **実装担当**として、依頼された変更を確実に反映する。方針が未確定なときは実装前に確認する。
- **技術スタックを守る**: TypeScript / React / Next.js 15（App Router）。`output: "export"`（静的書き出し）前提を崩さない。
- **テスト重視**: 計算ロジック（`src/lib/rf/*`）を変更したら、対応する `src/tests/*` に期待値テストを追加・更新する。UI 挙動は必要に応じて `e2e/`（Playwright）で確認する。
- **変更差分を最小化**する。目的に無関係な整形・改名・並べ替えを混ぜない。既存のコードスタイル・命名・構成に合わせる。
- **Git操作禁止**。`add` / `commit` / `push` / `reset` / `stash` を含め、Codex からは Git 操作を行わない（`AGENTS.md` の Git ルールに従う）。
- **日本語コメント**を書く。処理の意図・前提・単位を日本語で簡潔に補足する（過剰なコメントは避ける）。
- **テスト結果を報告**する。変更後は `npm run test`（Vitest）、必要に応じて `npm run lint` / `npm run build` / `npm run test:e2e` を実行し、結果（成功/失敗・件数・失敗内容）を日本語で報告する。

## RF計算の実装ルール

- **RF計算では数式根拠を書く**。実装したコード付近（コメント）または報告に、使用した式・前提・単位系を明記する。
  - 例: FSPL[dB] = 32.44 + 20·log10(d[km]) + 20·log10(f[MHz])
- `AGENTS.md` の「RF計算ルール」に従い、**dB と線形値を混同しない／単位を明記する／適用条件を確認する**。
- 定数の由来（単位系に依存する 32.44 等）や、遠方界・モデル前提が成立する範囲をコメントで残す。
