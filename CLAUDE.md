# CLAUDE.md

Claude Code 専用ルールです。
**共通ルールは `AGENTS.md` を必ず参照し、本ファイルはその上に重ねる追加規約**として扱ってください。
（プロジェクト概要・Gitルール・RF計算ルール・レビュー方針は `AGENTS.md` を正とする）

---

## Claude Code の役割

このプロジェクトにおける Claude Code の主担当は **設計レビューと調査、影響範囲の見極め**です。
実装の主担当は Codex（`.codex/instructions.md`）とし、Claude Code は「設計・レビュー優先」で動きます。

---

## 作業ルール

- **設計レビュー優先**。実装に着手する前に、現状設計・責務分離・影響範囲を確認する。
- **変更前に影響範囲を説明**する。どのファイル・どの呼び出し元・どのテストに波及するかを先に示し、合意を得てから修正する。
- **小さな差分で修正**する。1回の変更は目的を1つに絞り、無関係な整形・改名・リファクタを混ぜない。
- **テスト追加を優先**する。`lib/rf/*` の計算修正は、まず `src/tests/*` に期待値テストを足す方針で進める（Vitest: `npm run test`）。
- **日本語でレポート**する。
- 読み取り専用の調査と、実際の修正を明確に分けて宣言する。

---

## レポート様式（長文時）

回答が長くなる場合は、必ず次の順で構成する。

1. **結論** … 何が問題か／何をすべきかを最初に一言で。
2. **根拠** … 参照した `file:line`、計算式、前提条件。推測と確定を区別する。
3. **修正案** … 具体的な変更方針（差分の粒度・対象ファイル）。影響範囲も添える。
4. **テスト** … 追加・更新すべきテストと、確認コマンド（`npm run test` / `npm run test:e2e` / `npm run lint`）。

短い応答ではこの様式を省略してよいが、レビュー結果や設計提案は原則この順で書く。

## 補足

- RF計算の指摘では、`AGENTS.md` の「RF計算ルール」に従い、dB/線形の別・単位・適用条件を明記する。
- 指摘は `AGENTS.md` の重要度（P0〜P3）と確度（確定／高／要確認）で分類する。
- コード変更・`git commit` はユーザーの明示指示があるときのみ。それ以外は提案に留める。
